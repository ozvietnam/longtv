<h3>Luồng vận hành</h3>
<pre><code>Khai vấn tháng mới (ngày 1–3)
  → Chốt 1–2 mục tiêu P0
  → Bóc task → 5 phòng ban
  → Ghi TODO.md mỗi phòng
  → Lắp assignee
  → Thực thi + cập nhật status trên web
  → Review tháng (ngày 25–30)
</code></pre>
<p><strong>Web all-in:</strong> Vừa bàn định hướng, vừa xây kế hoạch, vừa thực thi — tất cả trên <a href="https://github.com/ozvietnam/longtv">kho tri thức LongTV</a>.</p>
<hr>
<h2>Nguyên tắc phụ (vận hành)</h2>
<ul>
<li><strong>Một nguồn sự thật:</strong> Git repo = kế hoạch + thực thi; web chỉ là giao diện đọc/cập nhật.</li>
<li><strong>Content viết ở root repo:</strong> Research, decisions, roadmap, clarifications đều viết ở root; <code>web/content/</code> chỉ là bản mirror tự sync trước build.</li>
<li><strong>Không làm việc ngoài khung V1–V7:</strong> Mọi việc phải map về 1 tầng V (tránh lan man). Xem <a href="/docs/01-project-structure/00-MACRO_PROBLEMS">Cây vấn đề lớn</a>.</li>
<li><strong>Chốt trước, làm sau:</strong> Tháng mới bắt đầu bằng khai vấn (30–60 phút), không nhảy thẳng vào làm.</li>
<li><strong>Done = có bằng chứng:</strong> Mỗi task <code>done</code> phải có link/file/kết quả đính kèm ở cột Evidence.</li>
<li><strong>Plug-and-play nhân sự:</strong> Template task đủ rõ để người mới đọc → làm ngay, không cần briefing 2 giờ.</li>
</ul>
